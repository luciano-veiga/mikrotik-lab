🌉 Bridge — Switch Virtual

📘 O QUE E BRIDGE?

Bridge une interfaces como um switch virtual. Portas LAN (ether2-4) e wlan1 entram na bridge. O IP do router vai na bridge, nao nas portas individuais.

/interface bridge add name=bridge1
/interface bridge port add bridge=bridge1 interface=ether2

🏷️ VLAN — Segmentacao de Redes

📘 O QUE SAO VLANs?

VLANs isolam segmentos numa mesma infraestrutura fisica. Ex: VLAN10 = Funcionarios, VLAN20 = Cameras, VLAN30 = Visitantes. Cada VLAN tem seu proprio IP, DHCP e firewall controlando o acesso entre elas.

/interface vlan add name=vlan10 vlan-id=10 interface=bridge1
/ip address add address=10.10.10.1/24 interface=vlan10
/interface bridge vlan add bridge=bridge1 vlan-ids=10 tagged=ether2 untagged=ether3

🔗 PPPoE — Cliente e Servidor

📘 PPPOE NO BRASIL

PPPoE e o protocolo de autenticacao usado pela maioria dos provedores brasileiros. O router cria uma conexao virtual usando login/senha do provedor. O IP publico e recebido dinamicamente.

/interface pppoe-client add name=pppoe-out1 interface=ether1 user=login@isp password=senha add-default-route=yes use-peer-dns=yes

📋 IP → DHCP Server

📘 DHCP SERVER

O DHCP distribui IPs automaticamente. Cada VLAN/segmento precisa do seu proprio DHCP. Configure pool, gateway e DNS.

/ip pool add name=pool-vlan10 ranges=10.10.10.10-10.10.10.254
/ip dhcp-server add name=dhcp-vlan10 interface=vlan10 address-pool=pool-vlan10
/ip dhcp-server network add address=10.10.10.0/24 gateway=10.10.10.1 dns-server=8.8.8.8,1.1.1.1

🔐 VPN — L2TP/IPSec

📘 VPN L2TP/IPSEC NO MIKROTIK

Permite acesso remoto seguro a rede interna. L2TP cria o tunel, IPSec criptografa. Muito usado para home office e acesso remoto de tecnicos.

Fluxo: cliente remoto → internet → MikroTik (L2TP server) → rede interna

💡 HABILITAR L2TP SERVER

/interface l2tp-server server set enabled=yes use-ipsec=required ipsec-secret=ChaveSecreta123

💡 CRIAR USUARIO VPN

/ppp secret add name=tecnico01 password=SenhaVPN service=l2tp local-address=10.1.1.1 remote-address=10.1.1.2
⚠️ Use uma ipsec-secret forte (pre-shared key). Nunca use senhas fracas em VPN — e a porta de entrada da sua rede!
⚡ QoS — Filas e Controle de Banda

📘 QoS NO MIKROTIK

Simple Queue e simples e rapido para limitar por IP. Queue Tree e mais avancado com PCQ (Per Connection Queue) para distribuir banda igualmente entre todos os usuarios.

/queue simple add name=cliente1 target=192.168.88.10 max-limit=10M/20M
/queue type add name=pcq-down kind=pcq pcq-classifier=dst-address

🔒 Seguranca — Hardening MikroTik

1. TROQUE A SENHA DO ADMIN IMEDIATAMENTE

/user set admin password=SenhaForte@2024!

2. CRIE USUARIO DEDICADO E DESATIVE ADMIN

/user add name=gerente group=full password=OutraSenha@123
/user disable admin

3. DESATIVE SERVICOS DESNECESSARIOS

/ip service disable telnet,ftp,www,api,api-ssl
Mantenha apenas ssh (porta 22) e winbox (porta 8291)

4. RESTRINJA ACESSO POR IP DE ORIGEM

/ip service set winbox address=192.168.88.0/24
/ip service set ssh address=192.168.88.0/24

5. DESATIVE NEIGHBOR DISCOVERY NA WAN

/ip neighbor discovery-settings set discover-interface-list=LAN

6. DESATIVE BANDWIDTH TEST E PROXY

/tool bandwidth-server set enabled=no
/ip proxy set enabled=no

7. MANTENHA ROUTEROS ATUALIZADO

/system package update check-for-updates
/system package update install

8. CONFIGURE NTP PARA LOGS PRECISOS

/system ntp client set enabled=yes servers=pool.ntp.org

9. EXPORTE BACKUP REGULARMENTE

/export file=backup-config
/system backup save name=backup-completo
⚠️ Regra de ouro: NUNCA deixe Winbox ou SSH acessivel pela internet (WAN)!
